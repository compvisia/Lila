# Contributing guidelines

Thank you for considering contributing to Lila!
Contributions of any size are highly appreciated!

## Code of Conduct

By contributing to Lila, you are required to adhere to the [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

- Reporting bugs (open an issue)
- Suggesting features (open an issue)
- Fixing bugs
- Implementing features

### Pull request process

1. Fork the repository
2. Create a new branch (`feat/...` or `fix/...`)
3. Make your changes
4. Open a pull request with a clear description

## Development

Follow the setup instructions in the [README.md](README.md) to build and run the project.

General guidelines:

- Test your changes before submitting a pull request
- Keep changes small and focused
- Prefer clear, readable C++23 code over cleverness
- Try to follow the existing coding style (a formal guide will be added soon™)

## Questions?

If you have any questions or suggestions, feel free to reach out using the [Discussions](https://github.com/compvisia/Lila/discussions) tab, or join our [Matrix community](https://matrix.to/#/#lila:matrix.org).

## One last thing!

Most importantly, have fun contributing!
